XBSL – это инструментарий для проектов **1С:Элемент**, написанных парами `Имя.yaml`
(описание элемента) и `Имя.xbsl` (модуль с кодом). Он даёт быструю локальную обратную
связь до медленной серверной компиляции при деплое – единственной проверки, которую
предоставляет сама платформа. Проект поставляется как движок на Python и расширение
VS Code поверх него.

## Что внутри

- **Линтер с автоисправлениями** – 97 правил в четырёх ярусах: структура yaml, соглашения
  по тексту и типографике, структура кода и семантика, проверяемая по данным платформы и
  самому проекту.
- **LSP-сервер** – живая диагностика, переход к определению и автодополнение для любого
  редактора с поддержкой LSP.
- **Скаффолдинг метаданных** – создание объектов, реквизитов, маршрутов и форм без ручного
  написания yaml.
- **Поиск по документации** – локальный полнотекстовый индекс, собранный из вашего
  дистрибутива 1С:Элемент.
- **MCP-сервер** – линтинг, поиск по документации и каждая операция скаффолдинга,
  доступные как инструменты для ИИ-агентов.
- **[Расширение VS Code](https://github.com/keyfire/xbsl/blob/main/editors/vscode/README.md)**
  (издатель `keyfire`, идентификатор расширения `keyfire.xbsl`) – подсветка синтаксиса,
  диагностика по всему проекту, конструктор форм, дерево метаданных и кнопка деплоя – всё на
  том же движке.

## Рядом

Инструментарий 1С:Элемент – это две половины одной петли, а третий инструмент делает
то же самое на соседней платформе.

- **[Elemctl](https://docs.keyfire.ru/elemctl/ru/)** – доставка того, что проверил линтер: сборка из
  исходников, загрузка, применение и честная проверка, что стенд действительно поднялся на
  новой сборке. Команда *XBSL: деплой на стенд* в расширении VS Code зовёт именно его.
- **[EDT-Bridge](https://docs.keyfire.ru/edt-bridge/ru/)** – та же мысль на платформе 1С:Предприятие: мост
  MCP внутрь 1С:EDT, через который агент читает конфигурацию, правит модули и формы,
  собирает расширения и отлаживается.

Данные о языке (ключевые слова, каталог типов stdlib, метамодель конфигурации) генерируются
из вашего дистрибутива 1С:Элемент и не поставляются с проектом – см.
[Данные о языке](/ru/GUIDE#данные-о-языке) в руководстве.

Не аффилировано с 1С. "1C:Element", "1C:Fresh" и связанные наименования – товарные знаки
соответствующих правообладателей – см.
[NOTICE](https://github.com/keyfire/xbsl/blob/main/NOTICE).
