# Как дорабатывать xbsl

[English](CONTRIBUTING.md) · **Русский**

Спасибо за вклад. Ниже – минимум, чтобы добавить правило или обновить данные.

## Окружение

Требуется Python 3.10+. Данные о языке в репозиторий не входят – сначала сгенерируйте их из
своего дистрибутива 1С:Элемент (иначе линтер и часть тестов работать не будут):

```sh
python tools/extract_grammar.py   --dist "<каталог дистрибутива>"
python tools/extract_stdlib.py    --dist "<каталог дистрибутива>"
python tools/extract_metamodel.py --dist "<каталог дистрибутива>"

pip install -e ".[dev]"     # линтер + pytest + PyYAML
pytest                      # тесты (без данных зависящие от них пропускаются)
python -m xbsl <путь>   # прогон на исходниках
```

## Как добавить правило

1. Создайте модуль в `xbsl/rules/` (или дополните существующий).
2. Объявите функцию-правило и повесьте декоратор:

   ```python
   from xbsl.diagnostics import Diagnostic, Severity
   from xbsl.engine import SourceFile, rule

   @rule("группа/имя", "Короткий заголовок", "B", severity=Severity.WARNING)
   def моё_правило(source: SourceFile):
       if source.kind != "xbsl":
           return
       # ... вернуть/выдать Diagnostic(path, line, col, rule_id, severity, message)
   ```

   - `тир`: `A` структура/YAML, `B` текст/конвенции, `C` код, `D` семантика.
   - `scope="project"` – для кросс-файловых правил; тогда функция принимает `list[SourceFile]`.
   - `enabled_by_default=False` – если правило шумит на легаси (включается через `--select`).
   - Позиции строка/колонка – 1-индексированные. Для позиций используйте `xbsl.lexer.linemap`.

3. Подключите модуль в `xbsl/rules/__init__.py` (импорт регистрирует правило).
4. **Главное правило проекта:** прогоните на корпусе реального проекта и добейтесь **нуля ложных
   срабатываний**. Если правило массово срабатывает на существующем коде – делайте его `info` и
   выключенным по умолчанию, а не заставляйте всех править легаси.
5. Добавьте тест в `tests/` (образцы – в `tests/test_rules.py`).

Лексер и данные о языке/типах извлечены из самой платформы (грамматика Xtext/ANTLR и доки
дистрибутива), а не выдуманы – держитесь этого принципа: сверяйтесь с первоисточником.

## Данные под новую версию Элемента

Данные версионированы в `xbsl/data/element/<версия>/`. Чтобы добавить новую версию,
возьмите её дистрибутив и запустите экстракторы – версия определится сама:

```sh
python tools/extract_grammar.py   --dist "<каталог дистрибутива>"
python tools/extract_stdlib.py    --dist "<каталог дистрибутива>"
python tools/extract_metamodel.py --dist "<каталог дистрибутива>"
```

Вендорные файлы дистрибутива не коммитятся (кэш в `.refs/`) – только производные JSON.
